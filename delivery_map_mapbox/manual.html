RYS配送マップ 使い方マニュアル

配送・集荷の現場で使うための操作手順です。最終更新: 2026年5月26日

まず覚えること

毎日の基本の流れ 配送モードと集荷モード 配送先の追加 集荷リストの取り込み スポット集荷 スポット集荷の通知 文字サイズ ピン表示と固定位置 完了・未完了の操作 ルート並べ替え 翌日の集荷リセット 困ったとき

毎日の基本の流れ

1. アプリを開く 位置情報の許可が出たら許可してください。現在地ボタンや車用ルート提案で使います。
2. モードを確認する 左上付近の モード: 配送 / モード: 集荷 を確認します。メニューから切り替えできます。
3. 配送先・集荷先を準備する 配送は手入力・音声・伝票撮影で追加します。集荷はメニューの 集荷コースを選択 を使います。コースを開くと、定期集荷と現在反映済みのスポット集荷が一緒に表示されます。
4. 順番を整える 徒歩・自転車なら近い順、車なら道路優先ルートを使います。
5. 終わったら完了にする リストの ✓ を押すと完了です。通信が悪い場所で押した場合も、未同期として保存され、通信回復後に自動で再送されます。

配送モードと集荷モード

配送モード 単発の配送先を追加して回るためのモードです。配送先はこの端末のブラウザに保存されます。
集荷モード Googleスプレッドシートの集荷リストを取り込んで使うモードです。完了状態はスプレッドシートにも書き戻され、他の人の端末にも同期されます。

切り替えは、画面左上付近の モード ボタン、またはメニューの 配送モードに切替 / 集荷モードに切替 から行います。

左上の モード ボタンを押すと、現在が配送モードでも集荷モードでも、配送モード、小舟町店、浜町店 南、浜町店 北 を選べます。

配送先の追加

伝票撮影

  1. 📷 伝票撮影 を押します。
  2. 配送伝票やラベルの住所が読めるように撮影します。
  3. 写真の向きが横向きになっている場合は、プレビュー画面で 左に90° / 右に90° を押します。
  4. この向きで読み取り を押します。
  5. 読み取った住所を確認し、必要なら備考を直して追加します。

音声入力

  1. 🎤 音声 を押します。
  2. 住所をゆっくり話します。
  3. 認識された住所を確認して追加します。

手入力

  1. ✏️ 手入力 を押します。
  2. 住所と備考を入力します。
  3. 追加 を押します。
住所検索でピン位置がずれることがあります。配送先やスポット集荷は、リストの 📍 から地図上の正しい位置へ移動できます。

集荷リストの取り込み

  1. メニューを開きます。
  2. 集荷コースを選択 を押します。
  3. コース、または 配送モード を選びます。
  4. このコースを開く を押します。
集荷リストの id は定期集荷ピンの番号、company はピンのラベル、address は住所として取り込まれます。完了/未完了は collected だけで管理します。 checked は使いません。
小舟町店を選んだ場合は 小舟町店スポット、浜町店 南を選んだ場合は 浜町店 南スポット、浜町店 北を選んだ場合は 浜町店 北スポット のスポット集荷も一緒に取り込みます。

集荷リストでは、同じ建物内の訪問先を見分けやすくするため、会社名を太字で上段に、住所と時間・連絡事項を細字で下段に表示します。

定期集荷はコースごとに端末へ保存されます。2回目以降は保存済みの定期集荷ピンを先に表示し、スプレッドシートとの照合後、新規の集荷先または住所が変更された集荷先だけ住所検索を行います。

スポット集荷

スポット集荷はエコ配システムから自動取得し、コースごとのスポット用シートに保存されます。

小舟町店 小舟町店スポット を一緒に読み込みます。
浜町店 南 浜町店 南スポット を一緒に読み込みます。
浜町店 北 浜町店 北スポット を一緒に読み込みます。
  • GitHub Actions が7時台から17時台まで10分ごとにエコ配を確認します。
  • アプリを開いた時や集荷コースを開いた時にも、スポット集荷チェックが走ります。
  • 短時間に何度も開いた場合は、サーバー側で間隔を空けるため、毎回エコ配へアクセスするわけではありません。
  • 同じスポット集荷は同じIDで更新されるため、同じピンが重複して増えないようになっています。
  • スポット集荷のピン番号は時間帯コードで表示されます。例: S09、S13、S16。

キャンセルの表示

エコ配側でキャンセルになったスポット集荷は、完了済みと同じ扱いで表示されます。担当者名の欄には キャンセル と表示されます。

スポット集荷の通知

新しいスポット集荷、またはキャンセルが見つかった時に、選択したコースだけスマホへ通知できます。

最初にアプリを追加する

iPhone Safariでアプリを開き、共有ボタンから ホーム画面に追加 を選びます。ホーム画面の RYS配送マップ を開いて設定してください。iOS/iPadOS 16.4以降が必要です。
Android Chromeでアプリを開き、表示される アプリをインストール ボタン、またはChromeメニューの アプリをインストール / ホーム画面に追加 を選びます。

App StoreやGoogle Playから探す必要はありません。RYS配送マップのページをホーム画面へ追加して使います。

通知を有効にする

  1. ホーム画面に追加した RYS配送マップ を開き、メニューの 通知設定 を開きます。
  2. 通知を受けるコースを1つ以上選びます。複数コースも選べます。
  3. 有効化 / 更新 を押し、端末の通知許可を許可します。
新規の通知例 小舟町店: S16 新規1件
キャンセルの通知例 浜町店 南: S14 キャンセル1件
ロック画面の通知には、コース名・時間帯コード・件数だけを表示します。会社名、住所、電話番号は通知に表示しません。通知を押すと対象コースの集荷画面が開きます。
iPhoneでは、Safariの共有メニューからこのアプリをホーム画面に追加し、ホーム画面のアイコンから開いて通知を有効にしてください。iOS/iPadOS 16.4以降が必要です。

定期確認は7時台から17時台まで10分ごとです。反映から通知までは通常最大約10分です。通知を止める場合は 通知設定 の 解除 を押します。

文字サイズ

メニューの Aa 文字サイズ から、画面内の文字を 小、標準、大 の3段階で変更できます。

  • 選択するとすぐに地図上のピン表示、リスト、操作ボタンへ反映されます。
  • 選んだ大きさはスマホごとに保存され、次回起動時も維持されます。
  • 地図上のピン位置は変更されません。

ピン表示と固定位置

通常ピン 未完了の配送先・定期集荷です。
スポット集荷ピン 青色で表示され、番号部分に S09 などの集荷時間コードが出ます。
完了済みスポット集荷 緑色で表示されます。未完了の青色と区別しやすくしています。
黄色いピン 住所検索の精度が低く、位置が仮の可能性があるピンです。
2件 などのラベル 同じ場所、または近い場所に複数のピンがあることを表します。

定期集荷の固定位置

定期集荷は毎回同じ場所へ行くため、スプレッドシートの lat、lng を優先してピンを立てます。固定位置が入っている集荷先は、毎回住所検索をしません。端末に保存された位置は高速表示に使い、スプレッドシート側の固定位置が変更された場合は同期後に更新されます。

定期集荷の固定位置を変更するには管理者PINが必要です。通常の配送先やスポット集荷の位置修正には、この管理者PINは不要です。

完了・未完了の操作

  • リストの ✓ を押すと完了になります。
  • 完了済みの行で ↩ を押すと未完了に戻ります。
  • 集荷モードでは、完了時に担当者名が必要です。
  • 担当者名はメニューの 担当者名を設定 から保存できます。
  • 通信が悪くてスプレッドシートへ書き込めない場合は、行の左側に未同期の印が出ます。通信が回復すると自動で再送されます。
集荷モードの完了状態はスプレッドシートへ自動で書き戻されます。他の人の端末は定期的にスプレッドシートを読み直すため、同じ完了状態を共有できます。

ルート並べ替え

徒歩・自転車向け 近い順 現在地、または地図中心から近い順に並べます。一方通行や道路事情は考慮しません。
車向け 道路優先ルート Mapboxの道路ルートを使い、一方通行や車で走れる道を考慮して順番を提案します。最大で現在地 + 24件までです。
Google車向け 道路優先ルート Google Routes APIの道路ルートを使って順番を提案します。Mapbox版と結果を比べたい時に使えます。最大で現在地 + 24件までです。
  1. メニューを開きます。
  2. 用途に合わせて 徒歩・自転車向け 近い順、車向け 道路優先ルート、Google車向け 道路優先ルート のどれかを押します。
  3. 提案内容を確認します。
  4. 問題なければOKを押して並べ替えます。

翌日の集荷リセット

集荷は毎日同じコースを回るため、前日に完了した集荷先は翌日に自動で未完了へ戻ります。

  • 集荷モードを開いた時に、前日以前の完了分を自動でリセットします。
  • スプレッドシートの collected、collected_at、collected_by も空に戻します。
  • 当日中に完了したものは、その日のうちは勝手に戻しません。
  • スポット集荷用シートは日付で管理されます。日にちが変わると当日のスポット集荷だけを表示します。

手動で戻したい場合

  1. 集荷モードでメニューを開きます。
  2. 集荷完了をリセット を押します。
  3. 確認画面でOKを押します。

便利な表示機能

  • ◎ 現在地: GPSをONにして現在地を表示します。
  • 表示: 番地+建物: ピンのラベル表示を切り替えます。
  • 町丁目境界: 町丁目の境界線表示を切り替えます。
  • 番地表示: 地図上に番地ラベルを表示します。ズーム17以上で見えます。

メニューの表示

メニューは配送モードと集荷モードで表示内容が変わります。集荷モードでは集荷時間とID順を優先するため、近い順や車用ルート検索は表示しません。

配送モードで表示 ルート検索、完了分を削除、API使用状況など、配送先を追加して回る時に使う項目です。
集荷モードで表示 担当者名を設定、集荷完了をリセットなど、スプレッドシートと集荷完了を共有するための項目です。
どちらでも表示 Aa 文字サイズ と 通知設定 は配送モードと集荷モードのどちらからでも開けます。

困ったとき

現在地が出ない

ブラウザの位置情報許可を確認してください。屋内ではGPS精度が落ちることがあります。

住所の場所がずれる

リストの 📍 から、地図上の正しい場所へピンを移動してください。

定期集荷の固定位置を変更する場合は管理者PINが必要です。管理者PINが分からない場合は、管理者に位置修正を依頼してください。

集荷の完了がスプレッドシートに反映されない

担当者名が未設定の場合は先に設定してください。通信が悪い場所で完了を押した場合は、未同期として端末に保存され、通信回復後に自動で再送されます。

スポット集荷が表示されない

集荷コースをもう一度開いてください。アプリ起動時とコース選択時にスポット集荷チェックが走ります。短時間に連続して開いた場合は、サーバー側の制限により少し待ってから反映されることがあります。

キャンセルのスポット集荷が残っている

キャンセル分は削除せず、完了済み扱いとして表示します。担当者名の欄に キャンセル と出ていれば正常です。

通知が届かない

メニューの 通知設定 で通知状態と選択コースを確認してください。設定できない場合は、通知設定画面に表示される端末別のインストール手順に従い、ホーム画面のアプリから開き直してください。端末側で通知を拒否した場合は、端末の通知設定から許可を戻してください。

伝票写真が横向きになる

撮影後のプレビューで 左に90° または 右に90° を押し、正しい向きにしてから この向きで読み取り を押してください。

車用ルートが失敗する

配送先が25点以上ある場合は、24件以内に分けて検索してください。Google車向けルートだけ失敗する場合は、Google CloudでRoutes APIが有効になっているか、サーバーのAPIキー設定を確認してください。それ以外の場合は管理者に連絡してください。

画面が古いままに見える

ブラウザで開いている場合は、ページを下に引っ張って更新、またはブラウザの更新ボタンを使います。ホーム画面から開くアプリ版では、メニューの アプリを更新 を押してください。
