# シフト自動通知システム 引き継ぎ資料

## プロジェクト概要

**目的**: 毎日19時にGoogleスプレッドシートのシフト表を読み取り、翌日の出勤者をLINEグループへ自動通知する

**送信メッセージのイメージ**:
```
@All
5/7(木) メンバー

【終日 9:30〜20:30】
知念 ホセミゲルさん
美濃 祐斗さん

【AM 9:30〜14:00】
亀井 俊介さん

【PM 16:00〜20:30】
木村 友哉さん

各自確認次第 👍 でリアクションお願いします。
```

---

## 構成

```
Google スプレッドシート（シフト表）
        ↓ 毎日19時にトリガー
Google Apps Script（shift_notification.gs）
        ↓
LINE Messaging API
        ↓
LINEグループに @All メンション付きで送信
```

## スプレッドシートの構造

```
行1 : （各日の通し番号）
行2 : A列=「2026年5月シフト」タイトル、E列以降=日付（Date型: 5/1, 5/2...）
行3 : 時間帯ラベル（「9:30-14:00」「16:00-20:30」が交互に並ぶ）
行4〜: スタッフデータ（D列=氏名、E列以降=シフト内容）
```

**日付列の並び方**（4列ごとに1日）:
- 5/1 → E列(AM), F列(PM)
- 5/2 → I列(AM), J列(PM)
- 5/3 → M列(AM), N列(PM)
- ...（G・H列などは空白の区切り列）

**シフト値の例**:
- `矢口渡` `鹿島田W` `川崎①` → 出勤（エリア名が入る）
- 空白 → 非出勤
- `休み` → 公休（スクリプトで除外処理済み）

2. メニュー「**拡張機能**」→「**Apps Script**」をクリック
3. 「コード.gs」にスクリプトが入っている

## 残り作業

### ① @All メンションのテスト（最優先）

スクリプトを最新版（`shift_notification.gs`）に差し替え後、`sendShiftNotification` を実行してLINEグループで **@All がメンション（青字）になっているか**を確認する。

もし青字にならない場合の代替案:
- LINE Messaging API の仕様変更の可能性 → [LINE公式ドキュメント](https://developers.line.biz/ja/docs/messaging-api/message-types/) を確認
- `textV2` + `substitution` 方式で実装済み（`$all` が `@All` メンションに置換される）

### ② 毎日19時の自動トリガー設定

GASで以下の手順:

```
1. 左メニューの「時計アイコン（トリガー）」をクリック
2. 右下の「トリガーを追加」ボタン
3. 設定内容:
   - 実行する関数: sendShiftNotification
   - イベントのソース: 時間主導型
   - 時間ベースのトリガーのタイプ: 日付ベースのタイマー
   - 時刻: 午後7時〜8時
4. 「保存」をクリック
```

## スクリプトの関数一覧

| 関数名 | 用途 |
|--------|------|
| `sendShiftNotification()` | **本番用**: 翌日のシフトを読んでLINE送信 |
| `testSendToday()` | **テスト用**: 今日のシフト読み取り確認（送信しない） |
| `isStaffName(name)` | スタッフ名か判定（エリアコード・集計行を除外） |
| `dateMatches(cell, date)` | セルの日付と指定日が一致するか判定 |
| `sendLineMessage(body)` | LINE APIへの送信処理 |

## トラブルシューティング

### 「シートが見つかりません」エラー
→ `CONFIG.SHEET_NAME` が `大田区 シフト` になっているか確認

### 「翌日のシフトが見つかりません」
→ 翌日の日付がスプレッドシートの行2に存在しない（月をまたぐ場合など）

### LINE送信エラー（400/401）
→ `LINE_CHANNEL_TOKEN` が正しいか確認。トークンは LINE Developers コンソールの「Messaging API設定」→「チャンネルアクセストークン（長期）」で再発行可能

### @All がメンションにならない
→ `sendLineMessage` 関数内の `substitution` 方式を使用中。LINE API仕様の変更があった場合は [LINE公式ドキュメント](https://developers.line.biz/ja/docs/messaging-api/message-types/) を参照

### GASのタイムゾーンズレ
→ 日付比較は `Utilities.formatDate(date, Session.getScriptTimeZone(), 'M/d')` で JST 統一済み
